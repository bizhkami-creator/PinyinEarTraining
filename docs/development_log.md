# 開発ログ

## Day1

### 作業内容

- アプリ名を決定
- アプリコンセプトを整理
- MVP要件を整理
- 画面構成を整理
- README.mdを作成
- requirements.mdを作成
- SDK 37へのアップデートとビルドエラーの解消

### 決定事項

- アプリ名は「ピンイン耳トレ - HSK1級対応」
- 最初はHSK1級の約150語を対象にする
- 音声はAndroid TextToSpeechを使う
- 出題形式はピンイン＋声調の3択 / 5択
- ランキングはSharedPreferencesで保存する

## Day8

### 作業内容

- `MainActivity` において、CSVから全単語を読み込んだ後にシャッフルし、選択された問題数分だけ抽出するロジックを実装
- 抽出した単語リスト（`ArrayList<Word>`）を `QuizActivity` および `AnswerActivity` 間で Intent を使って引き継ぐように変更
- `QuizActivity` で、引き継がれたリストから `currentQuestion` に応じた単語を表示するように更新
- CSVの単語数が選択した問題数より少ない場合に、出題数を自動調整する処理を追加

### 決定事項

- 1プレイ中の重複を避けるため、開始時にリストを確定させ、Activity間で共有する
- データの引き継ぎには `putParcelableArrayListExtra` を使用する

### 次回作業

Day9では声調ベースの選択肢自動生成を実装し、クイズとしての完成度を高める。(完了)

- 選択肢のシャッフルとボタンへの反映

## Day9

### 作業内容

- `PinyinUtils` を作成し、正解ピンインから1声・2声・3声・4声・声調なしの5パターンのバリエーションを生成するロジックを実装
- 単音節および多音節（先頭音節のみ変更）の両方に対応
- `QuizActivity` において、生成されたバリエーションから3択または5択の選択肢をランダムに抽出し、シャッフルして表示する機能を実装
- 選択肢が正解ピンインの声調違いのみで構成されるようになり、クイズの難易度が適切になった

### 決定事項

- 多音節語については、現時点では先頭の母音の声調のみを変化させる仕様とする
- 選択肢は毎回シャッフルして表示位置を固定しない

## Day12

### 作業内容
